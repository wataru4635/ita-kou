## 残存差分の内訳と原因
- ナビ文字グリフのアンチエイリアス差（XDラスタ vs ブラウザ Noto Sans JP レンダリング）= 残差の主因。
- 区切り線・ナビバー左端の約2px水平オフセット（XD PNG はヘッダー bbox 切り抜きで、実装は1440幅右端揃えクロップのため）。
- 「お問い合わせ」英字：XD のタイポ "COPNTACT" を実装では正しく "CONTACT" としたため当該箇所のみ差分（意図的・仕様）。
- 白いロゴ領域は差分ゼロ（白文字ロゴ＝白背景で両者とも不可視。MV実装後に視認可能）。

## 計測条件
- 計測ツール: pixelmatch (threshold=0.1) + NCC（独自 compare.cjs / pngjs）
- PC: ビューポート 1440x900 → ヘッダー右端 1333x104 をXD bbox（0001）と比較
- SP: ビューポート 375 → ハンバーガー右端揃えで 329x44 をXD bbox（0002）と比較
- XD参照: xd-file/header/（PC 0001 / SP 0002）, scale=1, 白背景(#fff)合成
- 成果物: 差分=mcp-log/.tools/diff-pc.png・diff-sp.png / オーバーレイ=screenshots/overlay/header-{pc,sp}-overlay.png

## 更新（PC/SP 切替を md=768px へ変更）
- ユーザー指定により PCナビ⇄ハンバーガーの切替を **lg(1024) → md(min-width:768px)** に変更。
- リキッド root font-size（base/_base.scss）により、ナビ幅・ロゴを素の `rem()` で書くと 768px では約0.64倍に比例縮小して収まる（768px: cell 85.4 / contact 66.5 / logo 211.8、overflowX 0）。1200px以上でデザイン実寸（cell 133.5 / contact 104 / logo 331）。
- **再測定: 1440px の PC pixeldiff は 96.19% / NCC 0.9548 で変更前と同一（1440px のレンダリングは同値のため回帰なし）**。767px=ハンバーガー / 768px=PCナビ を確認。
