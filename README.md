# XD → HTML/SCSS 自動コーディング システム

XDデザインカンプをAI（Claude Code）で自動的にHTML/SCSSに変換するシステムです。
XDからエクスポートしたJSON + PNGデータを使って、デザイン情報の解析からコーディング、セルフレビューまでを自動実行します。

GitHubからテンプレートをダウンロードし、MCP Serverに接続するだけで使い始められます。
パイプラインのコアロジックはMCP Server経由で配信されるため、常に最新版が自動適用されます。

> **Figma版もあります**: Figmaを使っている場合は、Figma版（`figma/` フォルダ）を使用してください。Figma版はFigma MCPで直接デザインデータを取得するため、XDエクスポートの手間が不要です。

## セットアップ

> セットアップ（前提条件・ファイル配置・MCP接続・パーミッション設定）は **[ルートREADME](../README.md)** を参照してください。

### XDデータのエクスポート

XDからデザインデータをエクスポートし、プロジェクトルートの `xd-file/` に配置してください。

必要なファイル：
- `export_XXXX.json` — レイヤー情報（テキスト、座標、色など）
- 各アートボードのPNG画像 — レイアウト確認用

## カスタマイズ（初回のみ）

**テンプレートに含まれる `CODING_RULES.md` と `patterns/` は、作者（FLOCSS 環境）のコーディングスタイルに最適化されています。**
ただし、初回コーディング時に AI があなたの環境を自動検出するため、手動でのカスタマイズは最小限で済みます。

### 過去プロジェクトの配置（推奨）

`reference-projects/` フォルダに、過去にコーディングしたプロジェクトを1〜2個コピーしてください。

```
reference-projects/
├── project-a/     ← 過去プロジェクト1（フォルダごとコピー）
│   ├── src/
│   │   └── scss/
│   └── ...
└── project-b/     ← 過去プロジェクト2（任意）
```

- SCSS ファイルが含まれていれば OK
- node_modules/ は含めても含めなくても可（自動で除外されます）
- 「自分が一番きれいに書けたと思うコード」を選ぶのがコツです

### 初回コーディング時の自動検出

過去プロジェクトを配置した状態で初回コーディングを開始すると、工程3（実装計画）で AI が自動的に以下を実行します：

1. `reference-projects/` 内の SCSS を走査（なければ `src/sass/`, `src/scss/` を使用）
2. CSS記法を検出:
   - rem関数（`rem()`, `pxToRem()` 等）の名前・引数・定義場所
   - メディアクエリ mixin（`mq()`, `sp()` 等）の名前・ブレークポイント値
   - 命名規則（FLOCSS / BEM / 独自）
   - `@use` / `@import` パターン
   - ビルド環境（Gulp / Vite / webpack）
3. `docs/project-env.md` に検出結果を自動で書き込む
4. 過去プロジェクトから独自UIパターンを抽出し `docs/patterns-custom/` に保存
5. 以降のコーディングでは `project-env.md` の記法に合わせてコードを出力

**これにより、`patterns/` のマスターパターン（HTML/CSS構造の知識）はそのまま活かしつつ、rem関数やメディアクエリなどの記法はあなたのスタイルに自動変換されます。**

> `project-env.md` の検出結果は手動で修正しても構いません。自動検出が期待通りでない場合は直接編集してください。

## 使い方

### 基本の流れ

```
1. XDからJSON + PNGをエクスポートし、xd-file/ に配置
2. Claude Codeにコーディングを指示
3. AIが自動でコーディング
4. 完了報告を受け取る
```

### 最もシンプルな使い方

XDデータを `xd-file/` に配置した状態で、Claude Codeに以下のように指示するだけです：

```
xd-file/ にXDのエクスポートデータがあります。
PIPELINEに従ってコーディングをお願いします。
```

AIがMCPサーバーからパイプラインを自動取得し、工程を自動で実行します。

## 注意点・Tips

### 1. XDエクスポートデータの品質が重要

JSONにはテキスト情報（フォント、色、座標）が含まれますが、レイアウトの大まかな構造はPNG画像を見て判断します。エクスポート時にアートボード単位で書き出してください。

### 2. 画像は自分でダウンロードする（Figma版との違い）

**XD 版では画像の自動ダウンロードはありません。** Figma版はREST APIで画像を自動取得しますが、XD版はデザイン内の画像（写真・アイコン等）を自分でエクスポートする必要があります。

**画像の命名を効率化する方法：**

1. XD から画像を**名前は変えずにそのまま**ダウンロードし、プロジェクト内の画像フォルダに配置する
2. Claude Code に以下のように指示する：

```
XDの情報から画像の用途を確認して、フォルダ内の画像ファイルと一致させてください。
```

3. 続けて：

```
コーディングルールの命名規則に沿ってリネームしてください。
```

これで、XD からエクスポートした名前のままの画像を AI が用途を判定して適切にリネームしてくれます。1つ1つ手作業で名前をつける必要はありません。

### 3. PCのみ / PC+SP の指示

| 提供するデータ | 動作 |
|---|---|
| PCのXDデータのみ | PC実装 + 基本レスポンシブ（SPは経験則ベース） |
| PC + SP両方のデータ | PC実装 + XDデータ準拠のSP実装（ピクセルパーフェクト検証可能） |

SP（スマホ）のXDデータも渡すとより正確なレスポンシブ対応が可能です。

### 4. 個人知識ベース（agent-local.md）

`docs/agent-local.md` にはあなた専用の知識が自動蓄積されます。
AIがコーディング中に発見した落とし穴やパターンが記録され、次回以降の精度向上に活用されます。
手動で `/learn [内容]` と入力して知識を追加することもできます。

## トラブルシューティング

### 「auto-coding MCPに接続できません」と表示される

自動コーディング用のMCPサーバーに接続できていません。[ルートREADME](../README.md)のMCP接続設定を確認してください。

### コーディング精度が低い

- `docs/CODING_RULES.md` をあなたのスタイルにカスタマイズしたか確認してください
- XDエクスポートデータのJSONが正しく出力されているか確認してください

### JSONの値とデザインが合わない

XDエクスポートのJSON座標は絶対値（x, y）のため、AIがレイアウトを誤認する場合があります。PNG画像と合わせて確認し、必要に応じて指示を追加してください。
